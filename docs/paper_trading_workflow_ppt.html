Workflow Overview

运行流程主线

一个 market 从被发现到被结算，中间会经历多次 forecast、mark、adjust 和 exit 判断。

1

Market Scanner

发现 event / market，保存 latest market metadata 和盘口快照。

2

Evidence Freeze

记录 evidence_frozen_at，检索只允许使用预测前可用证据。

3

Forecast Run

输出 p_m、p_f、edge、confidence、reasoning。

4

Trader Policy

根据 edge、spread、liquidity、risk budget 决定 trade / skip / watch。

5

Execution Sim

用完整 CLOB depth 模拟成交，写 order 和 fill。

6

Position Loop

周期性 mark-to-market，必要时加仓、减仓、退出或等待结算。

Position Lifecycle

持仓是长周期对象

Open

从 fill 聚合成 position，记录成本、side、size、opening thesis。

Mark

定期刷新 order book，写 paper_marks，更新未实现 PnL。

Adjust

新证据或价格变化触发重新 forecast，再判断加仓、减仓或 hold。

Exit

按止盈、止损、edge 消失、接近截止日、流动性退化退出。

Settle

resolution 后写 settlement，计算最终 realized PnL。

长事件的重点

持仓期间的每次观点变化都要进入 ledger。这样才能区分 forecast 错误、policy 错误和 execution 错误。

GUI 展示

右侧 workbench 展示当前 position、latest mark、entry thesis、next due refresh。
